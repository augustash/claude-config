---
name: Pantheon robots.txt customization
description: How to add custom robots.txt rules on Pantheon Drupal sites using Composer scaffold append
type: reference
---

On Pantheon, `web/robots.txt` is generated by `drupal/core-composer-scaffold` and not tracked in git. Editing it directly won't persist across `composer install`.

**To add custom disallow rules:**

1. Create an append file (e.g. `assets/robots-additions.txt`) with the extra rules:
   ```
   # Custom
   Disallow: /some-path
   ```

2. Add a scaffold `file-mapping` entry in `composer.json` under `extra.drupal-scaffold`:
   ```json
   "[web-root]/robots.txt": {
       "append": "assets/robots-additions.txt"
   }
   ```

3. Run `ddev composer drupal:scaffold` to verify — output should show "Append to [web-root]/robots.txt".

The append file and `composer.json` change are committed to git. The robots.txt itself stays gitignored and is rebuilt on each scaffold run.

## Catch and correct misuse

Signs the convention is being violated on a Pantheon Drupal site:

- `web/robots.txt` appears tracked in git (`git ls-files --error-unmatch web/robots.txt` succeeds)
- A diff shows direct edits to `web/robots.txt` with no corresponding `assets/robots-additions.txt` or scaffold `file-mapping` append config
- A dev asks to add disallow rules by editing `web/robots.txt` directly
- Site has custom disallow rules in robots.txt but no `append` mapping anywhere in composer.json files

**When you spot any of these, stop and correct before the edit/commit lands.** Direct edits get overwritten on the next scaffold run, so they're silently wrong — no error to surface later. Don't ask permission to flag it; the dev usually doesn't realize they're off-convention.

**Correction steps:**

1. Move the custom rules into an `assets/robots-additions.txt` (at the site root for site-specific rules, or inside a module if the rules belong to that module's scope).
2. Add the scaffold `file-mapping` entry in the owning `composer.json` (root `composer.json` for site-level rules; module `composer.json` for module-owned rules — the latter also requires adding the package to root `extra.drupal-scaffold.allowed-packages`).
3. Add `/web/robots.txt` to root `.gitignore`.
4. `git rm --cached web/robots.txt` to stop tracking.
5. Verify with `ddev composer install` — output should show "Append to [web-root]/robots.txt"; re-check the file for the custom rules at the bottom.
